# Contributing

## Branching
- `main` protected, `develop` for staging.
- Features: `feat/<slug>`; fixes: `fix/<slug>`.

## Commits
Use **Conventional Commits** (`feat:`, `fix:`, `chore:`, `docs:`).

## Pull Requests
- Link an Issue.
- Checklist passes: build, lint, tests.
- 1 reviewer required (Jim or Maurice).

## Local Setup
- Frontend: Next.js suggested.
- Backend: FastAPI/Express suggested.
